1. Ghosh 1.9

2. Calculate the total time required to transfer a 1000-KB file in the following cases, assuming an RTT of 50ms, a packet size of 1 KB data, and an initial 2×RTT of “handshaking” before data is sent:

  1. The bandwidth is 1.5 Mbps, and data packets can be sent continuously.
  2. The bandwidth is 1.5 Mbps, but after we finish sending each data packet we must wait one RTT before sending the next.
  3. The bandwidth is “infinite,” meaning that we take transmit time to be zero, and up to 20 packets can be sent per RTT.
  4. The bandwidth is infinite, and during the first RTT we can send one packet (21−1), during the second RTT we can send two packets (22−1), during the third we can send four (23−1), and so on.

Assume that 1 KB = 1024 bytes and 1.5 Mbps = 1,500,000 bits per second

3. Network programming

3.1 If you are not already familiar with UNIX network programming (via the POSIX sockets API), there are two resources that you can use to learn this topic.  First, there is an online guide called Beej’s Guide to Network Programming.  This guide is available in a number of languages, and you only need to focus on sections 2 through 6.  The later sections on data representation we’re going to cover later in the course when we cover remote procedure calls (RPCs).  The Beej guide includes some example server and client code that you are authorized to use in your homeworks/projects.

3.2 You can also take an online module (approx 3.5 hours long) which provides a tutorial on network programming.  This tutorial is free provided you’re registered for this course.  You may use the code presented in this video tutorial in your homeworks/projects.

4. Write a TCP client program in C or C++ which connects to the host cse224.sysnet.ucsd.edu port 5555. Your client program should send to the server your PID as an ascii string followed by a carriage return followed by a newline (ensure that any letters in your PID are sent in upper case). The server will then return, via the socket, a piece of literature, followed by a keyword. In your homework, (1) identify the title of that piece of literature, (2) identify the keyword, and (3) put your code as a text listing in your submitted PDF.  Please do not reproduce the text of the literature in your submitted write-up (just put the name).

If you think that the sockets-based literature server has crashed or is for some reason unavailable, you can check its status at this webpage: https://stats.uptimerobot.com/O8zq4upGZ2

5. We will be covering XML-RPC during the first module of the course.  Once we’ve covered that topic, write a C or C++ based XML-RPC client program which connects to the host cse224.sysnet.ucsd.edu port 7777.  Your client program should send your PID (as a String object) to the server.  The server will return, via the XML-RPC protocol, a piece of literature, followed by a keyword.  To figure out what method you should call on the server, use the ‘xmlrpc’ command to send a system command to the server that provides you with a list of available methods.  Then use that same command to get help on the arguments that the method requires.  In your homework, (1) identify the title of that piece of literature, (2) identify the keyword, and (3) put your code as a text listing in your submitted PDF.
